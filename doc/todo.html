  • think about changing the offical name from NAnt to Nant.
    Reason: Easier to type, no need to steal the popularity of Ant's name, nant has a name of its own.
  • think about moving all the code from the src/NAnt folder to a src/Nant/Core folder.
    Reason: currently the source folder does not map the files being created. Both nant.exe and nant.core.dll come from the same folder!
  • think about droping the SourceForge from the namespace identifier.
    Reason: most projects seem to be going that way and SourceForge really isn't the compnay that wrote it.
  • think about renaming FileSet xml elements: includes => include, excludes => exclude, name => pattern.
    Reason: name is vague in its purpose, includes and excludes are misleading as they should be singular
  • think about adompting camelCase for all Xml elements in the build file, eg, basedir would become baseDir.
    Reason: All .NET xml files are camelCase.
